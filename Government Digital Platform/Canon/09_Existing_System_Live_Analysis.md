> מסמך נלווה ל-`00_Canon_Project.md`. תיעוד **הליכה חיה במערכת האמיתית שאנו מעצבים מחדש** — אזור אישי + מנוע הטפסים של משרד התרבות והספורט / משרד המדע (פלטפורמת "DX Forms"). המיקוד הוא **שכבת ה-UX, התוכן והזרימות** — לא המימוש הטכני (היעד: Salesforce + מערכת העיצוב הממשלתית; הסטאק הקיים אינו רלוונטי).

## 1. מפת המסעות שנצפו בפועל

```
הזדהות (ריבוי אמצעים)
   ▼
MainPage = "בחרו במערכת הרצויה"  ← דף הבית בפועל (Launcher של מערכות)
   ▼  בחירת מערכת (כרטיס)
רשימת בקשות של המערכת (סטטוס כטאבים + מסננים)
   ├─ "+ הגשת בקשה חדשה" → תפריט סוגי בקשה → גייט "זיהוי נהג" → טופס (iframe)
   └─ כרטיס בקשה = תצוגתי בלבד (אין מסך פירוט/Timeline)
פרופיל (עריכת כתובת + פרטי קשר)
```

## 2. מה ראינו — מסך אחר מסך

### 2.1 הזדהות
- **ריבוי אמצעי הזדהות** (יותר ממה שהונח): הזדהות לאומית (ראשי) · **קוד אישי לדוא"ל** (ת.ז **או דרכון** — זהו נתיב "ללא ת.ז ישראלית / קטינים") · **רשת חברתית** (Google / Facebook / LinkedIn / Yahoo) · **כרטיס חכם** · קישור "מעבר לצורת התחברות הישן".
- **רקע תמונה דקורטיבי** (טאבלט/קפה, `MainImage.jpg`) — בדיוק הדקורציה ש-D-005 מסיר.
- חלון פתיחה ("מערכת חדשה לפניות משתמשים") + באנר תחזוקה + בורר שפה + FAB תמיכה.
- גופן **Rubik** (לא Assistant) — שינוי מחייב במעבר ל-IGDS.

### 2.2 דף הבית בפועל = Launcher מערכות (MainPage)
- כותרת **"בחרו במערכת הרצויה"**. רשת **כרטיסי מערכת**: *נהיגה ספורטיבית* (רשות לנהיגה ספורטיבית) · *הון אנושי* (מטה) · *נסיעות לחול* (מטה).
- כל כרטיס: **תמונה שיווקית דקורטיבית גדולה** (מכונית מרוץ / מזוודות / איור אנשים) + לוגו המשרד + תגית קטגוריה (מטה / רשות).
- **אין דשבורד מבוסס-משימה**: אין סטטיסטיקות, אין "מה דורש ממני פעולה", אין התראות — רק בחירת מערכת.
- סרגל עליון מינימלי: שם המשתמש + תפריט פרופיל (עריכת פרופיל · לתצוגה הקודמת · יציאה). קיים מושג **"תת מערכת"** בניווט.

### 2.3 רשימת בקשות (בתוך מערכת)
- כותרת מערכת כהה עם **תמונת רקע דקורטיבית** (מכונית מרוץ נשפכת ל-Header).
- **"+ הגשת בקשה חדשה"** → תפריט **סוגי בקשה** אמיתיים (Request Types תחת המערכת):
  נהג—רישיון חדש · חידוש רישיון · הוספת דרגה · הנפקה באובדן/גניבה · חידוש לאחר ביטול. → מאשר את שרשרת ה-Domain Model **System → Service → Request Type**.
- **מסננים**: סוג בקשה · **חיפוש חניך לפי מספר ת.ז** · טקסט חופשי · תאריך עדכון (פריסטים: החודש / 3 חודשים / השנה…).
- **סטטוס כטאבים** עם מונה (למשל "אושר 1") + קיבוץ לפי סטטוס. כלים: רענון · **הורדת רשימה** · סה"כ בקשות.
- **שורת בקשה (4 עמודות, תצוגתי בלבד)**: `מספר בקשה | ת.ז+שם חניך · סוג בקשה` · `תאריך עדכון` · `הערת סטטוס` · `מספר סימוכין` (מזהה נפרד, למשל בפורמט `RNWLC_…`). פס כחול בצד. **אין מסך פירוט בקשה ואין Status Timeline.**
- המשתמש כאן הוא **נציג ארגון** (רשות לנהיגה ספורטיבית) המנהל בקשות עבור **חניכים/נהגים** — מאשר את פרסונת "ארגון מגיש / נציג" (Domain §3.1) ואת מודל ניהול ההגשות המרובות.

### 2.4 פתיחת בקשה — גייט "זיהוי נהג"
- לפני הטופס: מודאל **"זיהוי נהג"** — `מספר זהות/דרכון *` + `מספר רישיון נהג *` → אישור. רק אז נטען הטופס.
- הטופס נטען ב-iframe (`WrappedPage.html`) עם **רקע דקורטיבי נוסף** (משפחה בשדה).
- (לא הוזנו פרטי נהג — מידע של צד שלישי; דפוס הכניסה תועד בלבד.)

### 2.5 פרופיל
- עריכה אפשרית: **כתובת למשלוח** (ישוב, רחוב, בית, דירה, כניסה, מיקוד) + **פרטי קשר** (טלפון ראשי, סלולרי) + "קבל עדכונים למייל". (פרטי הזיהוי מההזדהות — לקריאה.)

## 4. מה אומת מה-Canon (נכון בשטח)

- **System → Service → Request Type** — קיים ממש (מערכת "נהיגה ספורטיבית" עם 5 סוגי בקשה). ✓
- **Launcher של מערכות בצד האזרח/נציג** — קיים (MainPage). ✓ (מנהל ההחלטה D-010/D-011 מדויק.)
- **פרסונת "ארגון מגיש / נציג"** המנהל הגשות מרובות (חניכים) — אמיתית ומרכזית. ✓
- **ריבוי אמצעי הזדהות** + נתיב דרכון/קטינים — קיים. ✓ (להעשיר את `T-13`.)
- **פרופיל**: זהות מההזדהות לקריאה, קשר/כתובת לעריכה — תואם `T-20`. ✓

## 5. השלכות על הפרוטוטייפ (קלט לסבב התיקונים)

1. **דשבורד אזרח (T-01)** — לחזק כעוגן אמיתי מול ה-Launcher הקיים (לא להחליף את ה-Launcher אלא להוסיף שכבת "מה דורש פעולה").
2. **רשימת בקשות** — להוסיף Status Badge דו-רבדי לכל שורה, לצד שמירת קיבוץ/טאבים לפי סטטוס + מסנן "חיפוש חניך".
3. **מסך פירוט בקשה + Timeline (T-03)** — פער מרכזי שאין בקיים; ערך גבוה.
4. **Launcher מערכות (T-12/בחירת מערכת)** — לעצב מחדש את כרטיסי המערכת **ללא דקורציה שיווקית**, עם לוגו + שם + תגית + מונה בקשות.
5. **טופס (T-06)** — לשקף **שלב זיהוי מקדים** (כמו "זיהוי נהג") כשלב 0 מובנה, וקונטקסט "מגיש עבור חניך X".
6. **שפת Request Types** — לאמץ ניסוח קונקרטי בסגנון הקיים ("נהג — בקשה לחידוש רישיון").

## 6. ניתוח עיצובים קודמים (צילומי Figma — `איסוף חומרים/`) + **תיקון מיפוי מוצרים**

> ניתוח 10 מסכי Desktop + Mobile מהעיצובים הקודמים. **תיקון מהותי של הלקוח (17/06):** המערכת שנותחה (System Launcher + ניהול בקשות/חניכים) היא ה-**Government Operations Platform** — *ניהול הבקשות*. צד **משתמש הקצה (Citizen)** הוא נפרד וצריך להיראות **בדיוק כמו gov.il**.

### 6.1 ארכיטקטורת ה-Operations (כפי שבנויה בפועל)

| מסך | מבנה |
|---|---|
| **שער בחירת מערכת** (post-login) | "בחרו במערכת הרצויה" · מסך מלא · כרטיסי מערכת עם **מצב על הכרטיס**: פעיל · "המערכת מתעדכנת" (Maintenance) · "אין גישה — נדרש כרטיס חכם" (Locked). Header: אייקון App-Launcher + כותרת + משתמש. **זהו שער כניסה — לא פריט תפריט.** |
| **מעטפת בתוך מערכת** | Header כהה: שם מערכת + **"תת מערכת ▾"** (מחליף תת-מערכת) + לוגו משרד + **App-Launcher** (חזרה/החלפת מערכת) + משתמש ▾. **אין "בחירת מערכת" בתפריט צד.** |
| **רשימת בקשות** | "+ הגשת בקשה חדשה" · מסננים (מילת חיפוש · סוג בקשה · מתי עודכן) · "21 בקשות" · רענן · הורדת CSV · **אקורדיון מקובץ לפי סטטוס**: טיוטות (14) · השלמת פרטים (2) · בקשות התקבלו (1) · בקשות שאושרו (0). שורה: שם ותפקיד · תאריך עדכון · סטטוס · הערת סטטוס · **מספר סימוכין** · פעולות (✎ ⬇ ⋯). |
| **מסך טיפול בבקשה** | Header מירוץ + פס הקשר (שם · ת.ז · סוג בקשה · תג "חדש") + Action Bar (**בדיקה ושליחה** · טיוטא · מחיקה · חותמת זמן · "תוכן עניינים") + **מקטעים מתקפלים ממוספרים**: 1/4 טבלה (52,846) · 2/4 טופס · 3/4 טבלה ריקה (Empty) · 4/4 העלאת קבצים (תאריכים + Dropzones). |
| **Data Table** | מסך מלא: Header נייבי · עמודות רבות · Toolbar (חיפוש/סינון/מיון/נקה הכל/הורדת תבנית/בחירת עמודות/+ הוספת נתונים) · Checkbox לשורה. |
| **מצבי מערכת מלאי-עמוד** | Maintenance · Locked/No-Access · Loading (רקע כהה + ספינר "טוען..."). |

### 6.2 דקורציה להסרה (D-005)
תמונות מירוץ (Header), איור אנשים, DNA — בכל המסכים. **מוסרים** לטובת קו gov.il שקט; שומרים על **המבנה** (שער, אקורדיון סטטוס, מקטעים, מחליף מערכת).

### 6.3 תיקון ארכיטקטורה לפרוטוטייפ (מבטל את שגיאת סבב 1)
- **System Launcher הוא שער ה-Operations** (post-login), לא פריט תפריט ולא בצד האזרח. החלפת מערכת = דרך ה-Header (App-Launcher / "תת מערכת ▾"), כמו החלפת ארגון.
- **צד האזרח (Citizen) נבנה מחדש כמו gov.il** — דף שירות → טופס → אזור אישי my.gov.il. **בלי** Launcher מערכות.
- **שפה ויזואלית: נאמן ל-gov.il האמיתי** (D-018) — פלטת צבע, Header/Footer, כרטיסים ורכיבים בסגנון gov.il.
